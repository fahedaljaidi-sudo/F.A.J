To run this system permanently, the owner (client) must set up their own infrastructure accounts. This ensures they have full ownership and control over their data and costs.

### 1. Required Accounts
*   **GitHub (Free):** To host the source code.
*   **Railway ($5/month):** To host the Backend and PostgreSQL Database. (Requires a credit card for the Hobby plan).
*   **Cloudinary (Free):** To store patrol images and attachments permanently.
*   **Vercel (Free):** To host the Frontend (the website interface).

### 2. Configuration Steps
Once the accounts are created, the client must input these "Environment Variables" into the **Railway Dashboard**:

| Variable Name | Description | Where to get it |
|---|---|---|
| `DATABASE_URL` | The link to the PostgreSQL database | Auto-generated by Railway |
| `CLOUDINARY_URL` | The API link for image storage | Cloudinary Dashboard |
| `JWT_SECRET` | A secret code for login security | Create a random string of numbers/letters |

### 3. Estimated Monthly Costs
*   **Hosting & Database:** ~$5.00 USD / month (Railway).
*   **Image Storage:** $0.00 (Cloudinary Free Tier is sufficient for ~25,000 images).
*   **Frontend:** $0.00 (Vercel Free Tier).
